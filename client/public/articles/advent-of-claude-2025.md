## 入门指南

在深入了解功能之前，先设置 Claude Code 以了解你的项目。

### /init — 让 Claude 自动上手你的项目

每个人都需要入职文档。有了 `/init`，Claude 会自己编写。

Claude 会读取你的代码库并生成一个 `CLAUDE.md` 文件，其中包含：
- 构建和测试命令
- 关键目录及其用途
- 代码规范和模式
- 重要的架构决策

这是我在任何新项目中运行的第一件事。

对于较大的项目，您还可以创建一个 `.claude/rules/` 目录，用于存放模块化、特定主题的说明。该目录中的每个 `.md` 文件都会作为项目记忆与 `CLAUDE.md` 一起自动加载。您甚至可以使用 YAML frontmatter 根据文件路径有条件地应用规则：

```yaml
---
paths: src/api/**/*.ts
---

# API 开发规则

- 所有 API 端点必须包含输入验证
```

把 `CLAUDE.md` 看作是您的通用项目指南，而 `.claude/rules/` 则是针对测试、安全性、API 设计或任何其他值得单独列出的内容的重点补充。

### 记忆更新

想在不手动编辑 `CLAUDE.md` 的情况下保存一些东西到 Claude 的记忆中吗？过去，您可以用 `#` 开头，Claude 会将后面的内容追加到 `Claude.md` 文件的末尾。从 Claude Code 2.0.70 开始，情况不再如此，现在您可以直接告诉 Claude 为您更新 `Claude.md` 文件。

只需告诉 Claude 记住它：

> "更新 Claude.md：在这个项目中始终使用 bun 而不是 npm"

继续编码，不要打断您的心流。

### @ 提及 — 即时添加上下文

`@` 提及是给 Claude 提供上下文的最快方式：

- `@src/auth.ts` — 即时将文件添加到上下文
- `@src/components/` — 引用整个目录
- `@mcp:github` — 启用/禁用 MCP 服务器

在 git 仓库中，文件建议速度快约 3 倍，并支持模糊匹配。`@` 是从"我需要上下文"到"Claude 拥有上下文"的最快路径。

## 必备快捷键

这些是您经常使用的命令。把它们刻在肌肉记忆里。

### ! 前缀 — 即时运行 Bash

不要浪费 token 问"您能运行 git status 吗？"

只需输入 `!` 后跟您的 bash 命令：

```bash
! git status
! npm test
! ls -la src/
```

`!` 前缀会立即执行 bash 并将输出注入上下文。无需模型处理。无延迟。不浪费 token。不需要多个终端窗口。

这看起来很小，直到您意识到您每天要用它五十次。

### 双击 Esc 撤回

想尝试一种"如果我们……"的方法而不承诺它吗？

大胆尝试。如果变得奇怪，按两次 Esc 跳回到一个干净的检查点。

您可以撤回对话、代码更改，或两者兼而有之。这里要注意的一点是，运行的 Bash 命令无法撤消。

### Ctrl + R — 反向搜索

您过去的提示是可搜索的：

| 按键 | 动作 |
| :--- | :--- |
| Ctrl+R | 开始反向搜索 |
| Ctrl+R (再次) | 循环浏览匹配项 |
| Enter | 运行它 |
| Tab | 编辑第一个 |

不要重新输入。回忆。这也与斜杠命令无缝协作。

### 提示暂存 (Prompt Stashing)

就像 git stash，但是针对您的提示。

`Ctrl+S` 保存您的草稿。发送其他内容。当您准备好时，您的草稿会自动恢复。

不再复制到便签本。不再在对话中途打断思路。

### 提示建议

Claude 可以预测您接下来会问什么。

完成一项任务，有时您会看到一个灰色的后续建议出现：

| 按键 | 动作 |
| :--- | :--- |
| Tab | 接受并编辑 |
| Enter | 接受并立即运行 |

Tab 过去用于自动完成您的代码。现在它自动完成您的工作流。通过 `/config` 切换此功能。

## 会话管理

Claude Code 是一个持久的开发环境，将其优化到您的工作流中将允许您做更多事情。

### 继续您离开的地方

意外关闭了终端？笔记本电脑在任务中途没电了？没问题。

```bash
claude --continue    # 立即恢复您上次的对话
claude --resume      # 显示选择器以选择任何过去的会话
```

上下文得以保留。势头得以恢复。您的工作永远不会丢失。您还可以通过 `cleanupPeriodDays` 设置自定义会话保留多长时间。默认是 30 天，但您可以将其设置为任意时长，如果您不想保留 Claude Code 会话，甚至可以设置为 0。

### 命名会话

您的 git 分支有名字。您的 Claude 会话也应该有。

```bash
/rename api-migration       # 命名您当前的会话
/resume api-migration       # 按名称恢复
claude --resume api-migration  # 也可以从命令行工作
```

`/resume` 屏幕对分叉会话进行分组，并支持键盘快捷键：P 预览，R 重命名。

### Claude Code 远程

在 Web 上开始任务，在终端中完成：

```bash
# 在 claude.ai/code 上，启动一个 Claude Code 会话
# 它在您离开时在后台运行

# 稍后，从您的终端：
claude --teleport session_abc123
```

这会在本地拉取并恢复会话。无论在家还是在路上，Claude 都在。也可以通过 iOS 和 Android 的 Claude 移动应用程序以及 Claude 桌面应用程序工作。

### /export — 获取凭据

有时您需要记录发生了什么。

`/export` 将您的整个对话转储为 markdown：
- 您发送的每个提示
- Claude 给出的每个回复
- 每个工具调用及其输出

非常适合文档、培训或向过去的自己证明，是的，您确实已经尝试过那种方法。

## 生产力功能

这些功能消除了摩擦，帮助您更快地行动。

### Vim 模式

厌倦了伸手去拿鼠标编辑提示？

输入 `/vim` 并解锁完整的 vim 风格编辑：

| 命令 | 动作 |
| :--- | :--- |
| h j k l | 导航 |
| ciw | 更改单词 |
| dd | 删除行 |
| w b | 按单词跳转 |
| A | 在行尾追加 |

以思维的速度编辑提示。您数十年的 vim 使用肌肉记忆终于在 AI 工具中得到了回报。而且退出 Claude Code 的 vim 模式从未如此简单，只需再次输入 `/vim`。

### /statusline — 自定义您的视图

Claude Code 在终端底部有一个可自定义的状态栏。

`/statusline` 让您配置那里显示的内容：
- Git 分支和状态
- 当前模型
- Token 使用情况
- 上下文窗口百分比
- 自定义脚本

一目了然的信息意味着更少的中断去手动检查事情。

### /context — Token 的 X 射线视觉

想知道是什么吃掉了您的上下文窗口？

输入 `/context` 查看究竟是什么在消耗您的 token：
- 系统提示大小
- MCP 服务器提示
- 记忆文件 (CLAUDE.md)
- 加载的技能和 Agent
- 对话历史

当您的上下文开始填满时，这就是您弄清楚它去向的方法。

### /stats — 您的使用仪表板

```
2023: "看看我的 GitHub 贡献图"
2025: "看看我的 Claude Code 统计数据"
```

输入 `/stats` 查看您的使用模式、最喜欢的模型、使用连续记录等。

橙色是新的绿色。

### /usage — 了解您的限制

"我快要达到限制了吗？"

```bash
/usage        → 检查您当前的...
```
