# 45 天 Linux 生存挑战

## 导语：为什么在 2026 年自找麻烦？

说实话，我对这一年多前评测过的笔记本屏幕一直有怨念，直到这次厂商终于修复了它。现在的硬件堪称完美，本该是转向
MacBook 的最佳时机，除了一件事——Windows。为了搭建我的家庭实验室（Home
Lab），也为了在那台 Asus ProArt P16 上榨取每一分性能，我决定彻底抛弃舒适区。

从 11 月 24 日开始，我踏上了一段预计 30 天、最终延长到 45 天以上的 Linux
之旅。这段经历并非只有代码和极客的浪漫，更多的是 “崩溃与重建”
的循环。如果你想看一个 Windows
老用户如何因为一个休眠设置把背包烫得冒烟，或者如何在航班上对着黑屏怀疑人生，那么这篇文章就是为你准备的。这是一个关于选择发行版、解决驱动噩梦、并最终爱上这种掌控感的故事。

## 我的设备与起步：为什么选 Pop!_OS？

既然要折腾，就得用最强的硬件。我手里这台 P16 配备了 120Hz OLED 屏幕、AMD
处理器，以及一张被限制在 130W 功耗的 RTX 5090 显卡。如此激进的硬件组合，对 Linux
的驱动支持是个巨大考验。

> [!NOTE]
> 我的 Linux 探索之旅始于这台高性能笔记本。

在众多发行版中，我选择了 Pop!_OS 作为起点。理由很简单：它对 Nvidia
显卡的支持开箱即用，并且自带了非常便捷的 “混合显卡模式”
切换功能。对于像我这样既需要强劲 GPU
渲染视频，又希望在咖啡厅里能省点电的用户来说，这似乎是唯一理性的选择。起初的一周出奇的顺利，我沉浸在挂载虚拟驱动器、配置本地服务器的快乐中，甚至觉得
“Linux 也不过如此”。

### 初学者的顿悟

直到我因为安装 Steam（Flatpak 版本）导致路径混乱，又因为不知道如何在 GUI
里以管理员身份打开文件夹而抓狂时，我才意识到那句老话是对的：“如果你觉得 Linux
难用，通常是你的技能（Skill Issue）问题。”

## 45 天里的 “坑” 与解法：崩溃时间线

这 45
天并非一帆风顺，尤其是在我开始带着它旅行和高强度工作后，真正的问题开始显现。

### 第 2 周：高空惊魂与休眠失效

在一次飞行中，我合上盖子以为它进入了睡眠模式。结果落地时，我从包里拿出的不是笔记本，而是一块
“烙铁”。Pop!_OS
的默认睡眠模式在某些硬件上完全失效，机器一直在全速空转，电池耗尽。 **教训：** 在
Linux 上，尤其是在未完全调教好的硬件上，永远不要相信 “睡眠”，直接关机最安全。

### 第 3 周：集成显卡 (iGPU) 的黑屏危机

为了测试续航，我尝试手动切换到仅使用 iGPU
模式。重启后，屏幕直接黑了。我以为我彻底搞坏了系统。 **解决：** 通过 AI
辅助排查，我最终通过编辑启动参数（Boot
Parameter）才强制系统正确加载驱动，重新找回了画面。

### 第 4 周：外接显示器的缩放割裂

当我回到工作室，试图连接 240Hz 外接显示器时，发现了 Linux 桌面环境的痛点：混合
DPI 缩放。笔记本屏幕（HiDPI）界面微小如蚂蚁，而外接屏幕却正常。Pop!_OS
在这方面的处理（尤其是 X11 下）非常让人抓狂，很难实现每个屏幕独立的完美缩放。

### 第 5 周：升级即毁灭 (BusyBox)

为了解决电源管理依赖包的报错，我冒险将系统升级到 24.04
版本。结果重启后直接掉进了 BusyBox 命令行界面。 **原因：** 似乎是升级过程中 UUID
变更或引导损坏。这成为了压死骆驼的最后一根稻草，促使我开始寻找新的发行版。

## 工具与工作流：如何把 Linux 真正用起来

尽管系统层面问题不断，但在应用层面，我构建了一套效率极高的工作流。这也是我能在
Linux 上坚持 45 天的核心原因。

### 多端协作：KDE Connect & LocalSend

这是我离不开 Linux 的理由之一。配合我的 S25 Ultra 手机，KDE Connect
实现了剪贴板无缝同步和文件秒传。需要传大文件时，LocalSend 则是 AirDrop
的完美替代品，稳定且快速。

### 键位映射神器：Keyd

Linux 对输入设备的底层控制力极强。通过
Keyd，我重映射了键盘层级。同一个旋钮，在普通模式下调音量，在 DaVinci Resolve
模式下则变成了时间轴滚动轮。这种定制化在 Windows 上很难实现。

此外，浏览器体验也得到了升级。我使用了 Opera Neon，这是一款自带 AI
的概念浏览器。在这个充斥着驱动调试、论坛爬贴的 45 天里，它的 “工作区”
功能帮了大忙——我可以把所有关于 “DaVinci 故障排除” 的标签页聚合在一起，用 AI
一键总结论坛上的解决方案。

## 发行版之路：Pop!_OS → CachyOS → Zorin OS

这 45 天也是一段不断寻找 “理想乡”
的旅程。我的发行版选择经历了一次过山车般的变迁。

| 发行版       | 体验/状态  | 主要优缺点                                                        |
| :----------- | :--------- | :---------------------------------------------------------------- |
| **Pop!_OS**  | 起步与挣扎 | 优点：Nvidia 驱动支持极佳。缺点：系统升级易崩，睡眠可能有 Bug。   |
| **CachyOS**  | 短暂尝试   | 基于 Arch，性能优化激进。但对于不想每天修系统的我来说，过于复杂。 |
| **Zorin OS** | 未来目标   | UI 最接近 Windows，极其稳定。作为下一个主力系统进行长期测试。     |

## 给后来者的 10 条建议

如果你想要尝试 Linux，这里有 10 条用血泪换来的建议：

1. **调整心态**：把 Linux 当作一项需要学习的 “技能”，而不是开箱即用的家电。
2. **备份是命**：在做任何分区调整或系统升级前，务必备份重要数据。
3. **显卡策略**：Nvidia 用户首选 Pop!_OS 或 Zorin，除非你想手动编译驱动。
4. **忘掉睡眠**：除非确定硬件完美支持，否则 “关机” 比 “睡眠” 更省心。
5. **缩放问题**：尽量使用分辨率和 DPI 接近的多屏组合，或者研究 Wayland 的设置。
6. **AI 是导师**：遇到报错，先把日志丢给 AI，比搜技术论坛效率高十倍。
7. **理解日志**：简单的 `journalctl` 命令能让你知道电脑为什么挂了。
8. **创作工具**：确认你依赖的软件（如 DaVinci）是否有 Linux 版，并准备好面对小
   Bug。
9. **键位映射**：尽早配置 Keyd，这会让你的操作效率倍增。
10. **别怕重装**：觉得当前发行版不顺手？果断换一个，Distro Hopping 是必经之路。

## 结语：痛苦但值得

回顾这 45
天，虽然我修电脑的时间可能比用电脑创作的时间还多，但我并不后悔。这种对设备的完全掌控感是
Windows 无法给予的。随着 2026 年的推进，我计划继续我的 Linux 之旅——下一站是
Zorin OS。
